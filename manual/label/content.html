内容标签 - 模板标签

适用范围
全站任意地方均可使用，当content标签作为全局标签时，必须指定内容ID。
只有在内容页可不指定ID，不指定ID时，默认为当前内容ID 
case 'content:': 
{
    $attr = $this->getAttrs($matches[4]);
    if(isset($attr['id'])) $id = $attr['id']; else $id = '$contents["id"]'; 
    return 'getContent('.$id.')) as $key=>$content){?>';
}
标签作用
用于调取指定内容页的详细信息
调用方式 
  
// 更多调用请看下面的可调用字段
{content:id=4}
    内容标题：[content:title]
    区域ID：[content:area_id]
    栏目ID：[content:sorts_id]
{/content}
id：内容页id,不指定则默认当前内容页id（需在内容页有效）

添加自定义字段
我们添加了常用的内容字段，但并不一定能满足你的开发需求。你可以非常轻松地添加自定义字段。
自定义字段可以使用len,style

1.在数据库content表添加自定义字段

推荐使用phpmyadmin或navicat for mysql对数据库进行操作

2.在模板里添加自定义字段表单项

模板位置：\app\admin\Views\admin\html\content_edit.html
此处我们使用是js动态回填，你不必为自定义字段赋值
后台表单也无需处理，系统会自动将新添加字段值保存到数据库
